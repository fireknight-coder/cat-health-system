# 演示脚本（端到端路径）

无后端时请设置 `VITE_USE_MOCK=true`（.env.development 已默认），运行 `npm run dev` 后按以下路径演示。

---

## 1. 流1：上报 → 审核

1. 打开首页，选择身份「居民」→ 进入系统（演示登录）。
2. 左侧/顶部进入「上报流浪猫」→ 点击「添加素材」→ 填写位置、时间、备注 → 提交上报。
3. 退出 → 选择身份「管理员」→ 进入系统。
4. 进入「上报审核」→ 在列表中点击某条「审核」→ 查看上报信息与 AI 相似猫 Top-K、健康评分。
5. 选择「老猫绑定」选一只猫 → 确认为老猫；或「确认为新猫并建档」；或填写驳回原因 → 驳回。

---

## 2. 流2：干预工单与锁定

1. 管理员进入「数据仪表盘」或从审核/猫档案进入某只猫。
2. 在猫档案页（需有入口：如从审核结果跳转，或从仪表盘「在库数量」等进入猫列表）— 当前可从 URL 直接访问 `/admin/cats/c1`。
3. 点击「发起干预/抓捕/送医」→ 生成工单。
4. 进入「干预工单」→ 点击某条「详情」→ 填写处理结果备注 → 结案。
5. 猫档案页应显示「正在处理中」时不可编辑、不可领养；结案后解除锁定。

---

## 3. 流3：领养申请与回访

1. 居民端进入「可领养」→ 选择一只猫「申请领养」→ 填写电话、备注 → 提交。
2. 管理员进入「领养申请管理」→ 查看列表（电话、状态等）；电话联系后可在详情页记录联系日志（需详情页与接口）。
3. 同意/拒绝：当前列表页可扩展操作按钮，调用同意/拒绝接口；同意后 Cat 为 ADOPTED，建立回访计划；回访完成后更新为 FOLLOW_UP_DONE。

---

## 4. 流4：项圈与定位

1. 居民端「我的宠物」→「绑定项圈」→ 填写宠物 ID、设备 ID → 确认绑定。
2. 「我的宠物」中进入某宠物「查看位置」→ 查看实时位置/历史轨迹与异常告警（依赖后端定位与告警接口）。

---

## 5. 流5：数据仪表盘

1. 管理员进入「数据仪表盘」。
2. 查看四块：规模与变化、健康与风险、闭环效率、时空分布（后两块可依赖后端聚合与地图）。

---

## 联调开关

- 使用真实后端：新建 `.env.local`，设置 `VITE_USE_MOCK=false`、`VITE_API_BASE=https://your-api.com`，重启 dev。
- 使用 Mock：不设置或 `VITE_USE_MOCK=true`，无需后端即可完成上述演示路径。
